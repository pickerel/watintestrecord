Functions

> See the list of available functions

A function fills the gap between what can be gathered from a user's mouse clicks and keyboard entries, providing a means to add blocks of user-defined code. When you are recording a script and right-click, a window pops up showing the available functions for that HTML element. Functions are specially formatted HTML files located in the "Functions" directory with the main application and ending in ".trf" for Test Recorder Function.

There are some important points to the function page:

  • Title - the page title is used as the tab caption, so you may want to keep it short
  • POST - make sure your form uses a post method, otherwise data will not be captured
  • Startup Directory - relative references will function properly for additional code, images, etc.
  • FunctionTitle - a hidden input used to uniquely identify the function
  • Code - a hidden input for the entire source to be inserted into the recorded script
  • Settings - a hidden div with information on when to show the function. Within this div is:
    • attributes - one or more entries "and"-ed against the selected HTML element. For instance, an attributes tag with a name of "tagName" and value of "a" will match against link tags.
    • languages - a list of languages this function applies to
  • Data - a hidden div inserted when the function is shown containing JSON data. Provided in this data is:
    • CodeLanguage - current code language for properly formatting output
    • tagName - tag name of the HTML element
    • attributes - a listing of the HTML element's attributes
    • innerText - innerText of the HTML element
